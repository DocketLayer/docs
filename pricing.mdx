## Billing principle

**You are never charged for a failed single query.** We capture payment only on 200 responses. For batch requests, the payment transaction is sized for the full batch upfront (`$0.99 Ã— N queries`). If all cases in a batch fail, no payment is settled; if some fail, the full batch amount is still charged â€” x402 payments are atomic and cannot be partially refunded.

## Truncated responses

When `context=full` returns a response that would exceed 2MB, DocketLayer trims `docket_history` to fit. The response includes a `meta.truncated: true` flag and a `truncation_details` object. Truncated responses are billable â€” the query succeeded, data was delivered.

## Payment mechanics

Payment flows through the x402 protocol. On your first request to a paid endpoint:

1. DocketLayer returns HTTP 402 with machine-readable payment details â€” recipient address, amount, currency
2. Your x402 client constructs a $0.99 USDC transaction on Solana, signs it, and retries with proof attached
3. DocketLayer verifies on-chain and returns the data

Your code does not handle this directly. The x402 client library manages the 402 challenge, transaction construction, and retry automatically.
